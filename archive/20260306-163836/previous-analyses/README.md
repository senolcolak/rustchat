# Previous Analyses

Store one folder per implementation iteration.

See `./previous-analyses/INDEX.md` for the consolidated summary and migrated legacy analysis documents.

Folder naming pattern:

- `YYYY-MM-DD-<topic-slug>`

Each folder should contain:

- `SUMMARY.md`
- `SERVER_FINDINGS.md`
- `MOBILE_FINDINGS.md`
- `GAP_PLAN.md`

Copy initial structure from `./previous-analyses/_TEMPLATE/`.

Legacy analyses are grouped in dated buckets, for example:

- `./previous-analyses/2026-02-07-api-v4-baseline/`
- `./previous-analyses/2026-02-07-mobile-client-requirements/`
- `./previous-analyses/2026-02-07-compat-audit-reports/`
